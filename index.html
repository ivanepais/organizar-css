Organizar CSS

Mantener grandes archivos de CSS puede ser un desafio, debemos aprender practicas para que se nos haga más facil.

Tips para mantener tu CSS ordenado

Algunas sugerencias para mantener las hojas de estilo organizadas y ordenadas.

¿Tu proyecto tiene un estilo de codificación de CSS propio?

Una guia de estilo de codificación sirve para que todo el equipo de desarrolle siga unas normas y sea todo consistente, para que cada programador no aplique sus preferencias individuales y sean confusas para los demas.

Un ejemplo son las reglas para los articulos de mozilla mdn.

Manten el código consistente

Establecer reglas o normas de codificación para un proyecto grupal o individual es importante para mantener la coherencia.

Se aplica al describir el código, usando los mismo nombres/nombre correspondiente a cada elemento, para las clases; colores en distintas secciones de la web, un metodo para el espaciado en el código que sea facil de leer con una cantidad de tabulaciones, espaciado, etc.

Hacer código CSS legible

Es recomendable que las reglas de CSS (selector de elemento, propiedad y valor: declaración) esten separadas por un salto de línea; cada parte debe estar en una nueva línea.

Comentarios en el CSS

Ayudan a los programadores futuros a entender el CSS y a los presentes a recordar el CSS.

Un tip es hacer un bloque de comentario antes de una sección lógica o de mucho desarrollo de código, mirando rapido o para buscarlas directamente (debemos usar ||).

No es necesario comentar cada cosa, ya que en CSS, gran parte se explican sola. Pero podemos comentar cuando se ha tomado una decisión por una razón en particular. Un ejemplo pueden ser propiedades de respaldo para navegadores antiguos.

Otro ejemplo puede ser si has usado un tutorial para lograr algo, agregar la url.

Crear secciones para CSS

En principio se crea una sección en la que se le dará un estilo comun para todos los elementos de HTML, predecible para todos los navegadores.

La segunda sección sería de utilitdad, puede tomar las clases principales de los elementos. Por ejemplo, para eliminar los puntos en los elementos de lista; así no aparecerán en la barra de navegación. En esta sección podría poner elementos que no entran en otras secciones.

Una tercera sección podría ser los elementos que integren el diseño basico del sitio, como el encabezado, la barra/menú de navegación, etc.

En la cuarta sección, agregamos las funciones especiales o cosas especificas de la página, como menús contextuales, componentes.

Al ordenar de esta manera, buscaremos rapidamente lo que queremos cambiar.

Evitar los selectores demasiados especificos

Porque evita que dupliquemos elementos o partes del CSS para aplicar las mismas reglas.

Ej: aplicar a un parrafo que está dentro de un articulo article.main p.box {border: 1px solid #ccc;} . Selecciona el elemento descendientes del primero.

Si despues queremos aplicar la misma regla a algo fuera de .main o que no sea el elemnto p, tenemos que duplicar el css usando otro selector con las mismas reglas.

Para eso, usamos una clase para aplicar una regla multiples veces. Son pocas veces que se usa un selector especifico para aplicar css.

Dividir hojas de estilo grandes en varias más pequeñas

En caso de que se tenga estilos muy diferentes para distintas partes del sitio, es posible que sea mejor tener una hoja de estilo que tenga reglas globales y hojas de estilo más pequeñas que tengan reglas más especificas para esas secciones. Parecido a crear secciones de CSS.

Las reglas de la cascada de CSS se aplican normalmente, sin importar la cantidad de hojas aplicadas a un HTML. Por ejemplo, si ocupamos gran cantidad de CSS para las cartas que describen los productos de una tienda, tiene sentido tenerlo en una hoja aparte. Ademas agiliza y facilita su mantenimiento.

Otras herramientas que podemos usar

Son herramientas desarrolladas por la comunidad para poder mantener grandes hojas de estilo.

Metodologias de CSS

En lugar de tener que crear nuestros propios estandares o reglas para escribir CSS, podemos usar los enfoques ya creados. Se tratan de guias para escribir CSS, con un enfoque muy estructurado para mantener la consistencia para tener ordenada la hoja de estilo.

OOCSS

Oriented Object CSS es un trabajo de Nicole Sullivan, la idea es separar su CSS en objetos reutilizables para usarlos en cualquier lugar que necesite el sitio. El ejemplo estandar de OOCSS es el patrón llamado The Media Object. El patrón que se repite en todos los sitios es un contenedor con una imagen o video u otro elemento de tamaño fijo en un lado y contenido descriptivo en el otro (generalmente ocupa la mayoria de espacio). Pueden ser comentarios (imagen de usuario pequeña y texto del comentario), listados, etc.

A traves de clases, vamos nombrando a los objetos que son cada parte de nuestro sitio y despues las combinamos con selectores.

BEM

Significa Modificador de elemento de bloque. En BEM un bloque es una entidad independiente, como un boton, un menú o un icono. Un elemento es algo como un elemenento de lista (li) o un titulo que está vinculado al bloque en que se encuentra. Un modificador es una bandera en un bloque (independiente) o elemento (descendiente/relativo a otro) que cambia el estilo y el comportamiento.

El código BEM se reconoce facilmente por el uso extensivo de guiones medios y guiones bajos para las clases de los elementos. Las clases adicionales utilizan la estricta nomenclatura BEM.

Se usa ampliamente en proyectos web grandes y muchas personas que ayudan o escriben codigo en diferentes sitios, usan la nomenclatura BEM sin mencionarla. En CSS Tricks hay guia basica sobre BEM.

Otras metodologias

Hay otros como SMACSS (Scalable and Modular Architecture), ITCSS, ACSS (Atomic CSS), al ser populares puedes encontrar guias en la web. Por otro lado, usarlo en proyectos pequeños no tiene mucho sentido.

Construir metodologias propias

Hay herramientas disponibles para los desarrolladores que les permite adoptar un enfoque más programatico para escribir CSS. Se denominan preprocesadores y posprocesadores.

Un pre-procesador ejecuta archivos sin procesar y los convierte en una hoja de estilo, mientras en un post-procesador toma la hoja de estilo terminada y la optimiza para que cargue más repitdo.

Estas herramientas requieren de complementos (scripts, plug-in) que puede instalar en su editor de codigo. El pre-procesador más popular es Sass, a continuación se detalálarn conceptos basicos. Si queire aprender más hay una guia de Sass disponible en su sitio web.

Funciones basicas de Sass

Son realmente útiles en términos de organización, incluso si no usa ninguna de las otras funciones de Sass.

Definir variales

Configurar propiedades de CSS en una variable (es un objeto que le debemos poner un nombre -relacionado- con los valores que contiene) para poder usar todos estos valores en cualquier lugar de CSS. La gran ventaja de una variable es que si necesitamos cambiar algun valor, lo hacemos desde ella y se actualiza en todos los lugares donde esté. Por ejemplo, una variable que contenga el color azul; pero ahora el proyecto necesita usar el color roja, solo cambiamos una vez y no todos los lugares donde hayamos puesto el color azul.

$base-color: red; .alert {border: 1px solid $base-color;}.

Una vez compilado el CSS, pone el valor en la variable y no su nombre.

.alert {border: 1px solid red;}

Aunque ya podemos usar variables de CSS de forma nativa.

Compilar la hoja de estilo

Al usa Sass se puede tener multiples hojas de estilo muy pequeñas o incluso una hoja de estilo por cada componente.

Se usa la función partials que se pueden compilar juntos un una o una pequeña cantidad de hojas de estilo para vincular al sitio web.
